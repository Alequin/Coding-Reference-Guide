Convert a Decimal Number To Hexadecimal

Hexadecimal is a numerical system with a base (radix) of 16, which means the system has 16 unique symbols to represent values. To compare the decimal system has a base of 10 and the binary system has a base of 2.

To convert a decimal to a hexadecimal number the following step should be followed.

  1. Find the modulo of the starting decimal number. This will be the first digit of the hexadecimal number.
  2. Once done divide the starting number by 16 and round the answer down to get the second decimal number.
  3. Repeat step 1 on the second decimal number to get the second digit of the hexadecimal value.
  4. Repeat step 2 on the second decimal number to get the third decimal number.
  5. Once the result of dividing the decimal number is 15 or less the conversion is done. The result of the last division is the final hexadecimal digit.
  6. Make sure to change any digits between 10 and 15 to their letter equivalents.

Note: To find the modulo without a computer use this. The example finds n % a = b
n / a = x (Round x down to nearest whole number)
n - (x*a) = b

Example: Number to convert - 3000

  • 3000 % 16 = 8
  • Hex conversion: 8
  • 3000 / 16 = 187
  • 187 % 16 = 11
  • Hex conversion: 11,8
  • 187 / 16 = 11 (Less than 16. Final digit has been found)
  • Hex conversion: 11,11,8
  • Hex conversion: BB8

Example: Number to convert - 9

  • 9 % 16 = 9
  • Hex conversion: 9
  • 9 / 16 = 0(Less than 16. Final digit has been found)
  • Hex conversion: 0,9
  • Hex conversion: 9

Example: Number to convert - 16

  • 16 % 16 = 0
  • Hex conversion: 0
  • 16/ 16 = 1(Less than 16. Final digit has been found)
  • Hex conversion: 1,0
  • Hex conversion: 10

Example: Number to convert - 2112

  • 2112 % 16 = 0
  • Hex conversion: 0
  • 2112 / 16 = 132
  • 132 % 16 = 4
  • Hex conversion: 4,0
  • 132 / 16 = 8 (Less than 16. Final digit has been found)
  • Hex conversion: 8,4,0
  • Hex conversion: 840

Binary Search

The binary search algorithm identifies the location of a value within a sorted array. It works by identifying the mid-point in the array and comparing the value in this position to the target value. The array is then split at the midpoint. If the midpoint value is less than the target value the left array is used, otherwise the right array is used. This process is repeated, resulting in smaller arrays each time until the target is located or the array is too small to contain the target.

Image from: Oxford Math Center - Searching and Sorting Arrays

The average time for this algorithm to complete is Θ(log(n)).
The worst case time is also Θ(log(n)).

Quick Sort

The quick sort is a comparison based divide and conquer sorting algorithm that makes use of recursion. The sort works by first selecting a value to be the pivot. All values are compared against the pivot and moved. Values less than or equal to the pivot are moved to the left side, while values greater than the pivot are moved to the right side. This results in two arrays and the pivot in its sorted position.

These two arrays and all subsequently produced arrays undergo the same process. Once all subarrays lengths are equal to the base case the initial array should be sorted.

Image from: tek marathon - Understanding quicksort algorithm

The average time for this algorithm to complete is Θ(n*log(n)).
The worst case time is also Θ(n2).

In practice quick sort is generally faster than merge sort.

Merge Sort

The merge sort is a comparison based divide and conquer sorting algorithm that makes use of recursion. The sort works by dividing the given array of values in half multiple times until the remaining arrays have a length equal to the base case (usually 1).

Once the initial array has been divided the maximum number of times the values contained in the individual arrays are sorted and merged, which results in increasingly larger sorted arrays. This is repeated until all arrays have been merged. At this point the initial array should be sorted.

Image from: Khan academy - Overview of merge sort

The average time for this algorithm to complete is Θ(n*log(n)).
The worst case time is also Θ(n*log(n)).
