Content Containers

Content containers are tags which define an area of content on a web page. The div tag is the standard for this purpose as it is generic and may be used in any situation.

HTML5 also offers a number of semantic tags. While there is no difference between these and the div tag the variable names of the semantic tags can improve the layout of a document and better describe its contents.

  • <div></div>
  • <header></header>
  • <nav></nav>
  • <section></section>
  • <article></article>
  • <aside></aside>
  • <footer></footer>
  • <details></details>
  • <summary></summary>

Forms

HTML forms are used to collect information from a user. The input from the form may be used by front end javascript scripts or back end system. Forms offer elements such as text fields, checkboxes, radio buttons and drop down menus.

An important attribute for tags invloved with forms is name. Name is a method of identification for each section.

<form name="my_form"></form>
The tag form is the outer most tag of a HTML form. It is not required but functionality may be lost without it.

<input type="text" name="text_input" />
<input type="password" name="password_text" />

The tag input can be used in a number of ways. Here the type attribute instructs the input element to accept text. If tpye is set to password all input will be hidden.

<textarea name="text_area" rows="8" cols="80"></textarea>

The textarea is for larger amount of text. The rows attribute sets the max line height of the area. The user can change the height but it can never be less than the given number of lines. The cols attribute sets the width of the of the textarea.

The checkbox input is for when there are multiple options a user may select from.

<input type="radio" name="radio" >radio 1</input>
<input type="radio" name="radio" checked="true">radio 2</input>
radio 1 radio 2
Radio buttons are similar to checkboxes. The only difference is that users may only select a single option. Radio buttons that are associated should be given the same value for the name attribute.

<select name="drop-list">
  <option value="option 1"></option>
  <option value="option 2" selected></option>
</select>

<select name="drop-list2" size="3">
  <option value="option 1"></option>
  <option value="option 2" selected></option>
</select>

The select and option tags are used to create a drop down list of selections. The attribute select (which has no value) is used to set the initially selected value. The attribute size decides the height of the list (a size of three reveals three options). If the size is less than the number of options a scroll bar will be provided.

<input type="submit" name="submit" value="submit button"/>
<input type="reset" name="reset" value="reset button"/>
<input type="button" name="button" value="button button"/>
<input type="image" name="image" src="../images/puppy.jpg"/>
Buttons in HTML forms can be created with the input tag and may be one of four types.
  • submit: The submit button will call the process defined by the action attribute, which belongs to the the form tag. It may also re-load the page.
  • reset: The reset button will set all elements within the same form as the button to their default state.
  • button: This has no pre defined function. Actions can be assigned with javascript.
  • image: This acts like a submit button but also shows an image using the 'src' attribute to provide a link.

Note: The button tag is an alternative to the examples above. Read more

Declaring Language

Stating the language of content provides developers with more control over web pages that are provided in multiple language and assists browsers and other applications to deliver and present content. The lang attribute should always be provided in the html tag to allow all other tags to inherit the value. If any other elements contain alternate languages their corresponding tags should include the lang attribute and the appropriate language.

<html lang="en">

A list of avalible language tags can be found at IANA Language Subtag Registry and Language Subtag Lookup .

The value of the language attribute can be made use of in css. View the language in css section for more details.

Make Anchor Tags open a New Tab

Using the attribute target and providing a values of _blank will cause the link to open in a new tab.

<a href="www.website.com" target="_blank">website</a>

This however will create a vulnerable in your website as the linked site gains access to your page and is able to change the web address. This enables attacks to replace your site on the users browser with that delivers malware or one that look exactly the same in an attempt to gather personal information. This attack is known as phishing.

How phishing is achieved: the newly open site gains limited access to your page via the JavaScript window.opener object. This is a read/write object that can be manipulate. Among its properties is window.opener.location, which can be changed, causing the browser to go to a new URL.
(Paragraph taken from thesitewizard.com)

Phishing may be avoided by including the attribute rel with a value of noopener or noreferrer but not all browsers support this and so some users will be at risk.

<a href="www.example.com" rel="noopener noreferrer" target="_blank">
  example.com
</a>

Link Tag (CSS)

The link tag is used to define a connection between an external document (most commonly a css file) and the HTML file. The tag should be placed in the head element and, when used with a css file, requires three attributes: rel, type and href.

<head>
  <link rel="stylesheet" type="text/css" href="style.css">
</head>

  • rel: Defines the relationship between the external document and the current HTML document.
  • type: This specifies the type of document that is been linked.
  • href: This is the path / URL to the desired document.

Basic HTML Layout

<!DOCTYPE html>
<html lang="en">
  <head>
    <title>Page title</title>
    <meta charset="utf-8"/>
  </head>
  <body>
  </body>
</html>

The code above is the basic layout of a HTML file which most other webpages will start from.

<!DOCTYPE html>: This defines the document type While this is not required as browsers will assume you are using the current standard HTML5 it should always be included to ensure the page works is the future when HTML5 is not the standard. Also omitting this tag will force browsers to work in quirks mode, which is used by browsers to emulate the functionality of old browsers.

<html lang="en"></html>: The HMTL tag is the root element (top level) in a HTML document and is used to inform the browser that all code found between these tags should be interpreted as HTML. The lang attribute is not required but is recommended as all other tags will inherit this (view section on lang)

<head></head>: This tag is used to provide information about the web page and instructions for the webpage.

<body></body>: This is when the content of a web page is located. There can only be one body tag.

<title></title>: This is the title of the page.

<meta charset="utf-8">: The meta tag defines the character encoding of the web page. The attribute must contain a standard IANA MIME name for character encodings. The most recommended one is UTF8.

  • Authors should not use ASCII-incompatible encodings to avoid security risk: browsers not supporting them may interpret harmless content as HTML. This happens with at least JIS_C6226-1983, JIS_X0212-1990, HZ-GB-2312, JOHAB, the ISO-2022 family, and the EBCDIC family. (ASCII-incompatible encodings are those that don't map the 8-bit code points 0x20 to 0x7E to the 0x0020 to 0x007E Unicode code points)
  • Authors must not use CESU-8, UTF-7, BOCU-1 and SCSU, because cross-site scripting attacks with these encodings have been demonstrated.
  • Authors should not use UTF-32 because not all HTML5 encoding algorithms can distinguish it from UTF-16.

What is HTML

HTML (Hyper Text Markup Language) is used to create to structure content for web pages.

A Markup language is a computer language that defines the presentation of raw text by surrounding text with delimiters (tags in HTML) that the computer can interperet.

Hyper Text is text on a computer which links to text elsewhere via links known as "hyperlinks".
