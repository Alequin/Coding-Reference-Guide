Declaring Language

Stating the language of content provides developers with more control over web pages that are provided in multiple language and assists browsers and other applications to deliver and present content. The lang attribute should always be provided in the html tag to allow all other tags to inherit the value. If any other elements contain alternate languages their corresponding tags should include the lang attribute and the appropriate language.

<html lang="en">

A list of avalible language tags can be found at IANA Language Subtag Registry and Language Subtag Lookup .

The value of the language attribute can be made use of in css. View the language in css section for more details.

(Sources: w3.org: Declaring language in HTML
w3.org: Why use the language attribute? )

Make Anchor Tags open a New Tab

Using the attribute target and providing a values of _blank will cause the link to open in a new tab.

<a href="www.website.com" target="_blank">website</a>

This however will create a vulnerable in your website as the linked site gains access to your page and is able to change the web address. This enables attacks to replace your site on the users browser with that delivers malware or one that look exactly the same in an attempt to gather personal information. This attack is known as phishing.

How phishing is achieved: the newly open site gains limited access to your page via the JavaScript window.opener object. This is a read/write object that can be manipulate. Among its properties is window.opener.location, which can be changed, causing the browser to go to a new URL.
(Paragraph taken from thesitewizard.com)

Phishing may be avoided by including the attribute rel with a value of noopener or noreferrer but not all browsers support this and so some users will be at risk.

<a href="www.example.com" rel="noopener noreferrer" target="_blank">
  example.com
</a>

HTML Entities

In order to use reserved html characters entities are used. Entities normally have two identifiers (a name and a number) either of which can be used, though the number version is more likely to be supported by more browsers.
(Note: all "-" characters are used to avoid actually typing the entity. They are not part of the entities)

&-#60; / &-lt == <
&-#62; / &-gt == >
&-#38; / &-amp == &
