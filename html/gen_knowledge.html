In OOP it is the processes of hiding all but the relevant data about an object in order to reduce complexity and increase efficiency.

Hash Data Structure

Hashes are a data structure which make use of key pair values. They use an array to store values and generate an index using hashing techniques.

Hashing converts the given key using the modulo operator and uses the resultant value as an index. If two keys produce the same result, which is known as a collision, the array is searched for the next empty position. This is known as linear probing / open addressing. Another method to overcome collisons is separate chaining, which is when a linked list is created at the array index and all values that have keys matching that index will be stored in the list.

Array Data Structure

An array is a data structure which contains a collection of elements, each identified by an index or key. The indecies of an array are numbers, where the first element is 0, which signifes the elements positional relation to the first element (element 1 is index 0, element 2 is index 1 ect...).

The first element in the array is known as the first address or foundation address and is used to find all other addresses. If objects which take up 4 bytes are stored in the array (a 32 bit integer for example) and the first element is at address 2000 other elements may be found using 2000 + 4 * i.

Memory addresses 2000 - 2003 will store the first the first object and the second object will start at memory address 2004. If the index 1 is used the second element will be returned as 2000 + 4 * 1 will equal 2004.

Floating Point

Floating point numbers are used to represent decimal numbers. A floating point number is one where the decimal point may be placed anywhere relative to the significant digits.

Floating point numbers work by making use of a significand and an exponent value.

  • Significand: This represents the digits in the number (this may be negative if the floating point number is negative).
  • Exponent: The exponent determines the offset of the decimal point in relation to the given significand (this may be negative depending on the direction the decimal point should move).

Floating point number are only precise up to a specific number of digits after the decimal point depending on the number of bits which make it up.

A 32 bit floating point number will always be precise up to 6 significant digits.

A 64 bit floating point number will always be precise up to 15 significant digits.

Integer

An integer is a numeric value without decimals. They may represent positive or negative values.

Short integers are those which are able to represent number between -2^16 and 2^16 (-65536 to 65536) or less. Long integers are those which are able to represent numbers between -2^32 and 2^32 (-4294967296 to 4294967296) or more.
