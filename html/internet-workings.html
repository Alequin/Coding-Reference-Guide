Basics of The Internet

The internet is supported by a number of different types of hardware such as routers, servers, cell phone towers, satellites, radios, and other devices.

These elements are connections. Some are end points such as the clients machine, where requests are made, and others may be servers, which respond to client requests and return data. The other elements are nodes which direct connections between the endpoints. All these elements are connected by transmission lines such as cables or wireless signals from satellites, towers or radios.

To ensure the correct machines are communicating with one another protocols are used to define how machines connect and transfer data. The two most important protocols are TCP (transmission control protocol) and IP (internet protocol). They are commonly seen grouped and referred to as TCP/IP . Without these protocol two computers would not be able to transfer data without a direct connection and a common language shared between both computers. IP addresses are provided by DHCP’s (dynamic host configuration protocol), which dynamically assign addresses and ensures your computer is correctly configured to work on the network.

When clients make requests the TCP/IP protocols are used. The request is sent to your ISP (internet service provider), which routes the request to other servers and eventually to a DNS (domain name server). The role of the DNS is to use the given domain name to find the IP address of the target server (this is known as name resolution). If a matching IP address is found the connection is directed to the target server, otherwise the request is sent to another DNS until a match is found.

Note: The DNS can be bypassed if a valid IP address is provided instead of a domain name.

When the server receives a request it responds by sending packets of data commonly ranging in size from 1kb – 1.5kbs though they can be 9kb (size depends on the hardware used and the IP version in use). These packets contain the information that is to be returned to the client. Multiple packets relating to the same response may take different paths in order to avoid congested routes and increase the overall transmission speed.

packets use headers and footers to describe their contents and how the data they hold fits with the data from other packets. They will take the path of least resistance back to the client in order to avoid congested routes. Multiple packets relating to the same response may take different paths.

When the client has received the packets the instructions in the header/footer and the protocols are used to assemble them into a complete document.
