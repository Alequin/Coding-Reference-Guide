URI, URL & URN

A URI (Uniform Resource Identifier) is an umbrella term which includes URL’s (Uniform Resource Locator) and URN’s (Uniform Resource Name).

A URN is a method of locating resources on the web using a globally unique name. As the name is unique its location is not required to access the resource. The identifier of a resource is assigned within a defined namespace to ensure the uniqueness of the given name.

A URL is the human readable address of a resource on the internet that includes the protocol used to access the data. While there are a number of possible protocols the commonly used ones are http:// and https://.

URL’s consists of an access protocol, the domain name and a path to the directory on the server.

https://github.com/Alequin/Programming-Reference-Guide

Access protocol: http://
Domain name: www.github.com
Directory path: /Alequin/Programming-Reference-Guide

Additionally a URL might include a query string or a fragment

A query string is used to pass a string to a program that runs when the site is requested. The start of the sting is defined by a question mark symbol (?).

http://example.com/over/there?name=ferret

A fragment is used to link to a location within the page. A hash symbol (#) followed by the ID of an element is a fragment and when used will place the top of the named element will be placed at the top of the screen. A good example of these can be seen on all the URL's produced by a google search

https://alequin.github.io/Programming-Reference-Guide/html/internet-workings.html#ip-address

IP address

All machines (wether they are nodes or end points) have an IP address. This unique value identifies each machine on the network. While most machines use a version of IP known as IPV4 the number of addresses using this system are running out. IPV4 uses four 32-bit decimal integers ranging from 0 – 255. This can produce 232 possible addresses. IPV6 is the most recent version which uses eight 128-bit hexadecimal integers ranging from 0 - FFFF (0 - 65535), which allows for 2128 possible addresses, and while adoption of this system is being encouraged there are compatibility issues between the two. As most of the internet is based on IVP4 many are employing techniques to prolong the life of IVP4 rather than switching to IVP6.

IVP4 addresses: 232 = 4,294,967,296
IVP6 addresses: 2128 = 340,282,366,920,938,463,463,374,607,431,768,211,456

Note: While the large numbers above do represent the number of possible addresses using both systems this does not represent the true number of useable addresses as many are reserved. For example a valid IVP6 address must start with a number containing four digits and beginning with a 2 or 3.

IVP4
69.89.31.226
A single number in the address is known as an octet and each one is separated by a dot. The numbers are presented to humans in decimal format but the computer converts them into 8-bit binary numbers.
IVP6
2002:4559:1FE2:0:0:0:4559:1FE2
A single number in the address is known as an hexlet and each one is separated by a colon. The numbers are presented to humans in hexadecimal format but the computer converts them into 16-bit binary numbers.

The Three zero's in this address can be removed to save character. They are replaced with two colon's to produce 2002:4559:1FE2::4559:1FE2.

Basics of The Internet

The internet is supported by a number of different types of hardware such as routers, servers, cell phone towers, satellites, radios, and other devices.

These elements are connections. Some are end points such as the clients machine, where requests are made, and others may be servers, which respond to client requests and return data. The other elements are nodes which direct connections between the endpoints. All these elements are connected by transmission lines such as cables or wireless signals from satellites, towers or radios.

To ensure the correct machines are communicating with one another protocols are used to define how machines connect and transfer data. The two most important protocols are TCP (transmission control protocol) and IP (internet protocol). They are commonly seen grouped and referred to as TCP/IP . Without these protocol two computers would not be able to transfer data without a direct connection and a common language shared between both computers. IP addresses are provided by DHCP’s (dynamic host configuration protocol), which dynamically assign addresses and ensures your computer is correctly configured to work on the network.

When clients make requests the TCP/IP protocols are used. The request is sent to your ISP (internet service provider), which routes the request to other servers and eventually to a DNS (domain name server). The role of the DNS is to use the given domain name to find the IP address of the target server (this is known as name resolution). If a matching IP address is found the connection is directed to the target server, otherwise the request is sent to another DNS until a match is found.

Note: The DNS can be bypassed if a valid IP address is provided instead of a domain name.

When the server receives a request it responds by sending packets of data commonly ranging in size from 1kb – 1.5kbs though they can be 9kb (size depends on the hardware used and the IP version in use). These packets contain the information that is to be returned to the client. Multiple packets relating to the same response may take different paths in order to avoid congested routes and increase the overall transmission speed.

packets use headers and footers to describe their contents and how the data they hold fits with the data from other packets. They will take the path of least resistance back to the client in order to avoid congested routes. Multiple packets relating to the same response may take different paths.

When the client has received the packets the instructions in the header/footer and the protocols are used to assemble them into a complete document.
