Access Modifiers

  • public: public exposes its values to all locations
  • private: private means the value is only usable within the current location. If an instantiated object is passed into a method on its own class private variable may be influenced within the method (example show at the end).
  • protected: protected is similar to private. The difference is subclasses of the current class may use values with protected level access.
  • internal: internal access exposes values to the current assembly.
  • protected internal: combines protected and internal so that only subclasses within the same assembly may access values.

Accessing private member by passing an intance into a class

public class MoreTest{

  private String str = "to test it my purpose";

  public void test(MoreTest input){
    input.str = "that is why i exist";
  }
}
