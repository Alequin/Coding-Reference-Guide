Relative Measurements

  • em: This measure will size content based on the elements base font size (commonly 16px). So 2em is equal to twice the base font size. The standard base can be altered by providing a measure in pixels in a parent of the target.

    When setting the width of an element with em it uses the elements own font-size. So if the font-size of the element is set to 2em while the base is 16px (resulting in a font-size of 32px) setting a width of 10em will result in a width of 320px.
    (base font-size * font-size) * width = total width
    (16*2)*10 = 320.

    p{
      font-size: 2em;
      width: 10em;
      /*font-size == 32px*/
      /*width == 320px*/
    }

    body{
      font-size: 60px;
    }

    p{
      font-size: 2em;
      width: 10em;
      /*font-size == 120px*/
      /*width == 1200px*/
    }

  • rem: This is the same as em only the base font size is taken from the route html element.

    Unlike em when setting the width of an element rem actually the root elements font-size.

    html{
      font-size: 60px;
    }

    p{
      width: 10rem;   font-size: 2em;
      /*font-size == 120px*/
      /*width == 600px*/
    }

  • Percentage: All elements may be sized using percentage values, which will set elements contents boxes to x% of its parent. Setting width and height with percentages won't influence the margin, border or padding of an element so overflow is possible and should be considered.
  • vw: Sets sizes relative to view port width.
  • vh: Sets sizes relative to view port height.
  • vmin: Sets sizes relative to view port smallest side.
  • vmax: Sets sizes relative to view port largest side.
