Observer Pattern

The observer pattern is used when a state change in a single object (the subject) influences multiple other object (observers). Observers can be added and removed from the list of objects which the subject updates on its state change.

A real world example of this is newspaper subscribers. The newspaper publisher is the subject and subscribers are the observers. If a person is subscribed to the paper they will receive a new copy when it is available. If someone wanted to start receiving updates from the publisher they can subscribe, which would make them an observer. Subscribers may also leave the paper. This would mean they no longer receive updated copies and are no longer observers.
