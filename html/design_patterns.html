Decorator Pattern

The purpose of the Decorator pattern (or Wrapper pattern) is to attach additional responsibilities and functionality to objects dynamically. It is an alternative to using inheritance and subclassing. The decorator patten uses composition composition.

Composition is the idea of objects containing references to instances of others which contain desirable functionality. The decoration features which make the pattern possible (reference to desirable object(s) or required methods) are normally defined in a separate inheritable section of code (interfaces, mixin).

The following file is a very basic example of the decorator patterns using pizza.

pizza-decorator.rb

In the given example the class PizzaBase is decorated by Sweetcorn and Chorizo. While Sweetcorn and Chorizo could have been written without inheriting from another class it ensures the correct features are included. The reference to the object to be decorated is created in the Toppings class (@pizza) and meaning that all classes that inherit from Topping are able to wrap an object.

This can be seen in the final method get_stonebake_with_sweetcorn_chorizo as the pizza base is wrapped by sweetcorn, which is then wrapped by chorizo.

Observer Pattern

The observer pattern is used when a state change in a single object (the subject) influences multiple other object (observers). Observers can be added and removed from the list of objects which the subject updates on its state change.

A real world example of this is newspaper subscribers. The newspaper publisher is the subject and subscribers are the observers. If a person is subscribed to the paper they will receive a new copy when it is available. If someone wanted to start receiving updates from the publisher they can subscribe, which would make them an observer. Subscribers may also leave the paper. This would mean they no longer receive updated copies and are no longer observers.
