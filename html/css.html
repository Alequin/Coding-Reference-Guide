Box Sizing Property

The box-sizing property controls the way the box model interprets sizes set for elements.

The default value for elements is content-box, which applies the full width to the content section only. The padding and borders will add additional width and height to the element.

The value border-box will cause elements to take the padding and borders sizes of an element and reduce the content size so that an elements full size is equal to the set size. Margin width and height are not taken into account.

.box-size-example{
  width: 300px; height: 150px;
  padding: 25px;
  background-color: blue;
  border: 25px solid black;
  font-weight: bold;
}

#bs001{
  box-sizing: border-box;
}

id = bs001
class = box-size-example
id = bs002

Display Property

The display property specifies the type of box an element should use and is a key method of controlling the layout of a page.

Possible values

  • display: block - Elements set to this will use as much horizontal space as possible and not allow other elements to sit next to them
  • display: inline - Elements set to this will allow other elements to sit inline with them if they are able to. These elements will only take up the space they require and will not change their size even when instructed to by CSS.
  • display: inline-block - Elements set to this will allow other elements to sit inline with them if they are able to and their size may be manipulated with CSS.
  • display: none - This will make the element disappear from the page. The space it occupied will be filled by other elements.
  • display: flex - An elements with a display property of flex becomes a block level flex container. The contents of a flex container will sit next to one another. The direction of the contents flow may be changed to vertical if the container has the property flex-direction set to column.

    The contents of a flex container only require a value for the dimension that is parallel to the flow. For example if flex-direction is set to row (this is the default state) then the contents heights will start equal to the containers height (the height may be changed). The width must be set.

    Another useful feature of flex is the property flex-grow. This allows the parallel dimension to be set proportionally. For example if a flex container held two div's both with a flex-grow of one they would both occupy 50% of the container. If the first was set to 2 and the second to 1 the first would occupy 33.3% of the container while the seconds would occupy 66.6%.

CSS Selectors

Selectors are used in css to identify which elements should be influenced by the instructions written between the curly braces.

Basic selectors

*{}
The asterisk symbol will select and style all elements on a page.

#example{}
This is an id selector. The example will style the element with an id of 'example'.

.example{}
This is a class selector. The example will style all elements with the class 'example'.

div{}
This is a type selector. The example will style all elements with the tag 'div'.

div, p, .example{}
Multiple elements may be targeted at the same time if each one is separated with a comma. The example will style all elements with the tags 'div' and 'p' and all elements with a class of 'example'.

div p{}
.container div p{}
This is a descendent selector. It is used to select elements which are nested inside other elements. The first example will select all 'p' elements that are children of 'div' elements. The second example targets all 'p' elements that are nested within 'div' elements which are then nested within any elements with the class 'container'.

Additional selectors

div > p{}
.container > p{}
This is a child selector and will style directly nested elements. In the first example all 'p' elements that are direct descendants of 'div' elements will be styled.

div + p{}
.container + p{}
This is an adjacent selector and will style a single element that come after another. In the first example the first 'p' elements that appears after a 'div' element is closed will be styled.

div ~ p{}
.container ~ p{}
This is known as a general sibling combinator and will style all elements that come after another. In the first example all 'p' elements that appear after a 'div' element is closed will be styled.

Note: There are many more selectors. For more view the sources for this section

Fonts

Using CSS the font of text may be altered using the property. font-family.

p{
  font-family: "Times New Roman"
}

I am text! Click me to change the font.

It is common to provide a number of font names as fallbacks. This is done to maximise compatibility as some browsers and operating systems may not support all fonts. If the first is not available the next will be used. Also if a font name involves white space its name must be wrapped in quotes.

font-family: "Times New Roman", Times, serif;
font-family: Arial, Helvetica, sans-serif;

In order to increase the chance of the selected text style being used a web safe font should be selected. These are fonts which overlap across multiple systems and browsers. Web safe fonts include...

  • Arial
  • Arial Black
  • Helvetica
  • Times New Roman
  • Times
  • Courier New
  • Courier
  • Verdana
  • Georgia
  • Palatino
  • Garamond
  • Bookman
  • Comic Sans MS
  • Trebuchet MS
  • Impact

Add Images to Elements

Images can be added to elements using the property "background-image" and providing a link to an image in the form "url(image-link.jpg)"

The element which the image is being added to is a div with a height of 100px and a width that matches the sections width.

#image{
  background-image: url(../images/puppy.jpg);
}

The image has been added to the element (which is a div) however it is too large for the element to contain it. The property "background-size" is used to control the image size. It can be given multiple values.

The repetition of images can be controlled using the property "background-repeat". This accepts a few values such as "no-repeat", "repeat", "repeat-x", "repeat-y", "initial" (this set the default setting) and "inherit" (this takes the setting from the parent).

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
}

Here the value no repeat has been used.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px;
}

Here the size has been given one value, which controls the width. The hight will be selected automatically to maintain scale.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px 100px;
}

Here two values have been given. Width is still set to 150px but now height is manually set to 100px.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px 100px;
  background-position: 300px 100px;
}

Here the property background position has been set. This provides control over the images position relative to the top left corner. The first number defines the horizontal movement and the seconds defines vertical movement.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px 100px;
  background-position: right bottom;
}

The property background position is used again here. Rather than providing numbers for the horizontal and vertical positions however the terms right and bottom are used. This moves the image to the location describe by the terms (the right bottom corner). In addition to right and bottom the terms left and top may be used.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: contain;
}

In this example the word contain is provided, which causes the image to always fit within the element even if there is extra space.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: cover;
}

In this example the word cover is provided, which causes the image to always cover all the space of the element even when it does not fit.

Language in css

The value of the language attribute can be used by css to set styles based around the current language. Providing styles for multiple languages ensures text will always be ascetically pleasing regardless of language.

There are three main methods of controlling style in css based on language.

:lang() pseudo-class selector
[lang |= ""] selector that matches the beginning of the value of a language attribute
[lang = ""] selector that exactly matches the value of a language attribute

The word related is used a number of times in the following examples. This is to define exactly what is meant by related.

en is related to en, en-es, en-es-gct
es is not related to en, en-es, en-es-gct
en-es is related to en-es, en-es-gct
en-es is not related to en

The first example effects all related elements, whether the lang attribute was inherited or specifically stated.

p:lang(en){color: red;}

<html lang="en"><br>
...
<p>test 1</p>
<p lang="en">test 2</p>
<p lang="en-es">test 3</p>
<p lang="en-es-gct">test 4</p>

Vertically Align Text

div {
  height: 200px;
  line-height: 200px;
  text-align: center;

  border: 2px dashed black;
}
span {
  display: inline-block;
  vertical-align: middle;
  line-height: normal;
}

<div>
  <span>
    example text
  </span>
</div>

HyperText Markup Language (HTML) is the standard markup language for creating web pages and web applications. With Cascading Style Sheets (CSS), and JavaScript, it forms a triad of cornerstone technologies for the World Wide Web.

The span element is vertically centered by setting the div's line height equal to the div's height. The span is then set to an inline-block so that vertical align can be set to middle. The line height is then returned to normal.
The inner block does not have to be a span (though it may not work with all tags) as long as the display is set to inline-block.

!important priority

color: red !important;

Using the !important rule will cause the involved attribute to be considered as the highest priority and applied. While this can be useful is some cases it may lead to issues

Abuse of the !important rule may lead to confusion when others attempt to alter your CSS in the future and are unable to. This might result in others removing the rule, which could cause issues and additional work, or may cause future developers to abuse the same rule in order to apply their own code.
