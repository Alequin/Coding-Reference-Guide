Add Images to Elements

Images can be added to elements using the property "background-image" and providing a link to an image in the form "url(image-link.jpg)"

The element which the image is being added to is a div with a height of 100px and a width that matches the sections width.

#image{
  background-image: url(../images/puppy.jpg);
}

The image has been added to the element (which is a div) however it is too large for the element to contain it. The property "background-size" is used to control the image size. It can be given multiple values.

The reptition of images can be controlled using the property "background-repeat". This accepts a few values such as "no-repeat", "repeat", "repeat-x", "repeat-y", "initial" (this set the default setting) and "inherit" (this takes the setting from the parent).

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
}

Here the value no repeat has been used.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px;
}

Here the size has been given one value, which controls the width. The hight will be selected automatically to maintain scale.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: 150px 100px;
}

Here two values have been given. Width is still set to 150px but now height is manually set to 100px.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: contain;
}

In this example the word contain is provided, which causes the image to always fit within the element even if there is extra space.

#image{
  background-image: url(../images/puppy.jpg);
  background-repeat: no-repeat;
  background-size: cover;
}

In this example the word cover is provided, which causes the image to always cover all the space of the element even when it does not fit.

Language in css

The value of the language attribute can be used by css to set styles based around the current language. Providing styles for multiple languages ensures text will always be ascetically pleasing regardless of language.

There are three main methods of controling style in css based on language.

:lang() pseudo-class selector
[lang |= ""] selector that matches the beginning of the value of a language attribute
[lang = ""] selector that exactly matches the value of a language attribute

The word related is used a number of times in the following examples. This is to define exactly what is meant by related.

en is related to en, en-es, en-es-gct
es is not related to en, en-es, en-es-gct
en-es is related to en-es, en-es-gct
en-es is not related to en

The first example effects all related elements, whether the lang attribute was inherited or specifically stated.

p:lang(en){color: red;}

<html lang="en"><br>
...
<p>test 1</p>
<p lang="en">test 2</p>
<p lang="en-es">test 3</p>
<p lang="en-es-gct">test 4</p>

Vertically Align Text

div {
  height: 200px;
  line-height: 200px;
  text-align: center;

  border: 2px dashed black;
}
span {
  display: inline-block;
  vertical-align: middle;
  line-height: normal;
}

<div>
  <span>
    example text
  </span>
</div>

HyperText Markup Language (HTML) is the standard markup language for creating web pages and web applications. With Cascading Style Sheets (CSS), and JavaScript, it forms a triad of cornerstone technologies for the World Wide Web.

The span element is vertically centered by setting the div's line height equal to the div's height. The span is then set to an inline-block so that vertical align can be set to middle. The line height is then returned to normal.
The inner block does not have to be a span (though it may not work with all tags) as long as the display is set to inline-block.

!important priority

color: red !important;

Using the !important rule will cause the involved attribute to be considered as the highest priority and applied. While this can be useful is some cases it may lead to issues

Abuse of the !important rule may lead to confusion when others attempt to alter your CSS in the future and are unable to. This might result in others removing the rule, which could cause issues and additional work, or may cause future developers to abuse the same rule in order to apply their own code.
(Source and additional information: CSS tricks - When Using !important is The Right Choice )
