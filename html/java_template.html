Exceptions and Error Handling

Exception allow code to manage uncontrollable situations and coding mistakes effectivly. When an exception arises the normal flow is disrupted and the program will exit abnormally.

In Java the class Throwable is the superclass for all classes which are involved in error management. The three main subclasses of Throwable are Error, Exception and RuntimeException.

Error is used in situations in which a program should not recover from. They are abnormal conditions that are not commonly encountered and are beyond the programmers control.
Exception (also known as checked exceptions) and its subclasses (except for RuntimeException and its subclasses) are used for anticipated issues which a program could recover from. These occure at compile time and must be handled by the programmer.
RuntimeException (also known as unchecked exceptions) and its subclasses are used in general situations such as the array index being out of bounds.

Common runtime / unchecked exceptions

  • ArrayIndexOutOfBoundsException: Array index is out-of-bounds.
  • ClassCastException: Invalid cast.
  • IllegalArgumentException: Illegal argument used to invoke a method.
  • IndexOutOfBoundsException: Some type of index is out-of-bounds.
  • NullPointerException: Invalid use of a null reference.
  • NumberFormatException: Invalid conversion of a string to a numeric format.
  • UnsupportedOperationException: An unsupported operation was encountered.

Common Checked exceptions

  • IOException: Thrown due to issues with reading or writting files.
  • ClassNotFoundException: Class not found.
  • DataFormatException: Class not found.
  • NoSuchMethodException: Class not found.
