Base Constuctors

When inheriting from a super class the super classes constructor may be called from the sub classes using the keyword base.

public abstract class Mammal{

  private String voice;
  private String hair;

  public Mammal(String voice, String hair){
    this.voice = voice;
    this.hair = hair;
  }
}

public class Dog: Mammal{
  public Dog(String hair): base("woof", hair){}
}

Enum

Enums are sets of constants. The values will have an associated integer value (may use byte to long), which is accessed by casting the enum value. This value may be set manually.

If no integer is given the first enum will begin at 0 and each following value will have one greater value.

public enum EnumTest{
  Mon, Tue, Wed, Thu, Fri, Sat, Sun
}

Console.Write((int)Mon);
Console.Write((int)Tue);
Console.Write((int)Wed);
Console.Write((int)Thu);
//=> 0123

If the first enum is given a value the sequence will begin from the given number.

public enum EnumTest{
  Mon = 4, Tue, Wed, Thu, Fri, Sat, Sun
}

Console.Write((int)Mon);
Console.Write((int)Tue);
Console.Write((int)Wed);
Console.Write((int)Thu);
//=> 4567

Each time an enum is given a value the next value will 1 plus the previous (unless it is specifically set).

public enum EnumTest{
  Mon, Tue, Wed = 4, Thu, Fri, Sat, Sun
}

Console.Write((int)Mon);
Console.Write((int)Tue);
Console.Write((int)Wed);
Console.Write((int)Thu);
//=> 0145

Access Modifiers

  • public: public exposes its values to all locations
  • private: private means the value is only usable within the current location. If an instantiated object is passed into a method on its own class private variable may be influenced within the method (example show at the end).
  • protected: protected is similar to private. The difference is subclasses of the current class may use values with protected level access.
  • internal: internal access exposes values to the current assembly.
  • protected internal: combines protected and internal so that only subclasses within the same assembly may access values.

Accessing private member by passing an intance into a class

public class MoreTest{

  private String str = "to test it my purpose";

  public void test(MoreTest input){
    input.str = "that is why i exist";
  }
}

Output Parameters

Output parameters allow multiple values to be returned from a method using the parameters list rather than a return statement. If a method defines an out parameter if must be used like any other parameter.

public static int OutTest(int a, int b, out int c, out int d){
  c = a * a;
  d = b * b;
  return a + b;
}

int a = 0;
int b = 0;

int c = OutTest(2, 10, out a, out b);

Console.WriteLine(a);
Console.WriteLine(b);
Console.WriteLine(c);
//=> 4
100
12

As can be seen in the example using the out parameters does not mean a return statement cannot be used, though as always void may be used. All out parameters must be assigned though before the method ends.

Structure

A structure is a value data type which allows for the creation of simple Objects which may hold multiple properties. They can be useful as they require less memory that an object created from a class.

As Structures are value data types when they are passed into methods any alterations made by that method will not influence the source data (the same way the source int value is not changed when passed into and altered by a method).

Stucts should be defined either within the namespace or class blocks but outside any method blocks. They cannot inherit functionality from other structs or classes but may implement interfaces.

Below are a few examples of how structs may be used.

struct Building{

  public String name;
  public uint floors;

  public Building(String name, uint floors){
    this.name = name;
    this.floors = floors;
  }
};

Building skyScraper = new Building("Gurking", 150);
Console.WriteLine(skyScraper.name+" "+skyScraper.floors);
//=> Gurking 150

struct Animal{
  public String name;
  public uint age;
  public String type;

  public static Animal dog(String name){
    Animal dog;
    dog.name = name;
    dog.age = 0;
    dog.type = "dog";
    return dog;
  }

  public void logAnimal(){
    Console.WriteLine(this.name + " " + this.age + " " + this.type);
  }
};

Animal cat;

cat.name = "Billy";
cat.age = 4;
cat.type = "cat";
Console.WriteLine(cat.name +" "+ cat.age +" "+ cat.type);
cat.logAnimal();

Animal dog = Animal.dog("Teddy");

dog.logAnimal();
//=> Billy 4 cat
Billy 4 cat Teddy 0 dog

Nullable

A Nullable is a value type that is able to hold a null value;

int? = null;
bool? = null;

There is also a Nullable operator which uses two operands and the operator '??'.

b = null;
c = b ?? 20;
Console.WriteLine(c);

b = 30;
c = b ?? 20;
Console.WriteLine(c);

//=> 20
30

As shown in the example if the first operand is null the second will be used (null ?? 20 returns 20). If the first operand is not null it will be returned (30 ?? 20 returns 30);

Reference Types

Reference types do not contain actual data but the memory address to the required data. If a refence type is changed all varables that hold its memory location will refer to the changed value. Reference types in C# include object, dynamic and pointer.

  • Objects Types: This is the base type for all data types in C# Common Types System (CTS) and is an alias for the class 'System.Object'. The specific type of the object must be defined prior to compilation.
  • Dynamic Types: this type is able to hold any other type without declaring the what type is been stored. However when used it must be cast to it's required
  • Pointer Types: Pointer type variables store the memory address of another type.

Value Types

Value types directly contain the data they are given and are derived from the class 'System.ValueType'.

  • bool
    • A true or false boolean value
    • default: false
  • char
    • A 16 bit unicode character
    • default: '\0'
  • int
    • A 32 bit integer
    • -2,147,483,648 to 2,147,483,647
    • default: 0
  • decimal
    • A 128 bit precise decimal value with 28-29 significant digits
    • (-7.9 x 1028 to 7.9 x 1028) / 10^(0 to 28)
    • default: 0.0M
  • long
    • A 64 bit integer
    • -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
    • default: 0L
  • short
    • A 16 bit integer
    • -32,768 to 32,767
    • default: 0
  • sbyte
    • A 8 bit integer
    • -128 to 127
    • default: 0
  • double
    • A 64 floating point number
    • (+/-)5.0 x 10-324 to (+/-)1.7 x 10308
    • default: 0.0D
  • float
    • A 32 floating point number
    • -3.4 x 1038 to + 3.4 x 1038
    • default: 0.0F
  • ushort
    • A 16 bit integer
    • 0 to 65,535
    • default: 0
  • uint
    • A 32 bit integer
    • 0 to 4,294,967,295
    • default: 0
  • ulong
    • A 64 bit integer
    • 0 to 18,446,744,073,709,551,615
    • default: 0

Basic Structure

using System;
namespace HelloWorldApplication{

  class HelloWorld{

    static void Main(string[] args){
      Console.WriteLine("Hello World");
      Console.ReadKey();
    }
  }
}

  • using: the using keyword allows the code included in the provided namespace to be used in the current file.
  • namespace: this is where the namespace the current file is in should be defined.
  • class: this is the class declaration. A class contains the variables and methods that define the capabilities and behaviours of instances of that class.
  • static void Main(string[] args): this is the definition of the main method, which is the start point of all C# programs (the M in Main must be a capital).
