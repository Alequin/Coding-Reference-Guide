Removing Commits

To remove the previous commit use the command:

git reset --hard HEAD^

To remove one or more commits use the command:

git reset --hard HEAD~n
 (n is the number of commits to remove)

Rename Branch

Rename the current branch.
git branch -m newname

Rename a different branch.
git branch -m oldname newname

Common Commands: add

This command is used to stage new or modified files. When commit is called the staged files current states will be updated in the repository. Add may be called numerous times before the staged changes are commited.

git add file.txt
git add file.txt file1.txt
git add folder/file.txt
Add must always be followed by additional information to instruct git on what action to take. The simplest instruction is the path to the file to be staged (the full file path should be provided). As shown it is possible to provide multiple files to be staged.

git add *
Add may be used with a asterisk, which will identify all files in the current directory and stage their changes. As deleted files are no longer in the directory the deletion will not be staged.
Note: The asterisk is not part of git but is a globbing operator which returns all file names within the working directory. This can be used with commands such as ls * and is supported by most shells

git add .
Using a dot character after git add works in a similar way to the asterisk. The difference is it will add all files to the staging area even if they are deleted or hidden files.
Note: In my experience the previous paragraph is true but information on the web seems to say that git add . will not stage deleted files.
git add . vs git add *
Difference between “git add -A” and “git add .”

git add -u
git add --update
Using -u or --update with git add will stage all changed made to files but will start tracking any new files.

git add -A
git add --all
git add --no-igonore-removal
The command git add -A will stage all files.

git add 'command' -v
git add 'command' --verbose
The verbose command is used to print details on the actions that are taken when other commands are called. For example if the command 'git add -A' is called all files will be staged but the terminal will not report this until git status is called. If 'git add -A -v' is called the terminal will print out a summary of the results.
git add -p
git add --patch
The command -p enables you to selectivly stage changes with files rather than staging the entire file. This can be done on all files by calling the basic command (git add -p) or on specific files by providing a file name (git add file1.txt -p).

Additional options may be used with the command status. These can be viewed on the git-add documentation.

Common Commands: status

git status

This command displays files that have been altered or removed since the last commit. The first set of files are those that are staged and will be to be added or updated on the next commit.

When called the terminal will display the name of the current working branch, the number of commits the local repository is ahead of the remote repository (if a remote repository is connected), the staged changes that will be added on the next commit, unstaged changes and finally any untracked files.

Options that may be used with the command status include...

git status -s
git status --short
This produces a quick summary of the state of all the files in the directory. The state of each file is identified using single character: M = modified, A = added, D = deleted, R = renamed, C = copied, U = updated but unmerged, ? = untracked.

git status -s -b
git status -s --branch
This provides a summary of the current branch along with the summarised file states.

git status -v
git status --verbose
Along with the normal output of the command it produces the results of the command git diff (presents details of the changes made within each file)

Additional options may be used with the command status. These can be viewed on the git-status documentation.

Common Commands: init

git init

This command is used to create a new repository in the current working directory. A subdirectory named .git will be created in the directory, which is the basics required for a repository. No files contained in the directory will be tracked by git until they are added and committed.

This command may also be used to reinitialise a current repository.

Additional options may be used with the command init. These can be viewed on the git-init Manual Page.

What Is Git

Git is an open source version control system developed by the creator or Linux (Linus Torvalds). Version control is a method of recording changes to a set a files over a given period of time and allows the user to roll back the contents to a previous state if required. The details on all changes made are stored in a data structure known as a repository. A repository on the users machine is known as a local repository. If the files are hosted by a server, which allows for collaboration on a project, it is known as a remote repository.

Git is a distributed version control system (as opposed to a centralised version system) which means when clients checkout the files from a remote repository all of the changes are also provided, meaning the new user has access to the full file history.

Git works by recording snapshots of files. On each commit the state of each file that has changed is recorded. If a file has not changed a link to a previous version is provided to save space.
