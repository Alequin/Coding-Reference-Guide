Public, Protected and Private

In Ruby methods may be publically accessable, protected or private. Private methods are usable by the class that defines them and by subclasses. They cannot be called with the self keyword however. Protected are the same as private methods except that they can be called with the self keyword. Public methods are usable in allcases and by instances of the class or child class.

While methods can be set to public this is also their default state. There are a few ways they can be made public, protected or private.

Class methods can only be public or private and are set as shown in the next example file. Class methods are also public by default but can be specifically set to public.

Class method example

class MyClass_1

  def print
    public_print
    protected_print
    private_print
  end

  def public_print
    puts "Im public"
  end

  protected
  def protected_print
    puts "Im protected"
  end

  private
  def private_print
    puts "Im private"
  end

end

class MyClass_2 < MyClass_1
  def print
    public_print
    protected_print
    private_print
  end

  def print_2
    public_print
    self.protected_print
    self.private_print
  end
end

Using these classes it can be seen that the method print_public is usable in all situations.

a = MyClass_1.new()
b = MyClass_2.new()
a.public_print
b.public_print
#=> Im public
# Im public

The protected and private methods are usable by the print methods in both classes but cause errors when used by an instance.

a = MyClass_1.new()
b = MyClass_2.new()
a.print
b.print
a.protected_method OR a.private_print
#=>Im public
# Im protected
# Im private
# Im public
# Im protected
# Im private
# ERROR: protected method / private method (NoMethodError)

The private method is usable within MyClass_1 and MyClass_2 but causes errors when self.private_print is called. The protected method has no issues with being called with self.

b = MyClass_2.new()
b.print_2
#=>Im public
# Im protected
# ERROR: private method (NoMethodError)
