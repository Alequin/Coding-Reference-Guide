Redirection

Redirection is changing the input or output location of the terminal.

The symbol '>' is used to redirect the output of commands into a file. The file does not have to exist but if it does its contents will be overridden. In order to append the contents rather than override use the symbol '>>'.

pwd > file.txt
ls > file.txt
cat file1.txt > file2.txt
cat file 1.txt file2.txt >> file3.txt

To redirect the input the symbol < is used.

Another character that is used in redirection is pipe (|), which takes the output of the first command and uses it as input for the second command.

ls | sort
ls | grep work-file
cat file.txt | sort

Print and Search File Contents

cat file.txt
cat -n file.txt
This command stands for concatenate and is used to print the contents of files to the terminal. The option -n may be used to number each line.

less file.txt
more file.txt
These commands also print out the contents of the file to the terminal but will only prints as much as will fit in the terminal window. The user can then press enter to continue through the rest of the file. If less is used the print out of the file will be removed when the process is finished.

The less tag can also be used to search a document for a string. While viewing the document if '/search-term' is types (where search-term can be any string) the search term will be highlighted in the text (this is case sensitive).

head file.txt
tail file.txt
These commands print out the first (head) or last (tail) 10 lines of the file.

grep 'search-term' file.txt
egrep 'reg-exp' file.txt
grep -n 'search-term' file.txt
grep -i'search-term' file.txt
This command will print out all lines in the given file containing the search-term, which can be a regular expression if egrep is used. Using the option -n will print out the line numbers. Using the option -i will cause the search to ignore case. Note: wrapping the search term in quotes is not required for single words but if spaces or a regular expression are used they must be included.

wc file.txt
wc -c file.txt
wc -w file.txt
wc -l file.txt
These commands return the character, word and line counts of a document (in that order). The options may be used to select one of the three values. character count (-wc), word count (-w), line count (-l).

Basic Commands

man pwd
man clear
This command stands for manual and is used to show the documentation on a command inside the terminal.

.
..
The dot character represents the current directory and can be used with a number of commands. Two dots represents the parent directory.
*
*.jpg
The asterisk character is used as a wildcard and is used with commands in place of a file and directory name. Alone it represents all files and child directories in the current directory but it may be combine with other characters to match specific files, as shown in the examples.

The asterisk character actually represents any number of characters. Alone it matches to any file name that has any characters in it (which is all file names).

?
file.?xt
The question mark character is used as a wildcard to represent a single character.

pwd
This command stands for print working directory and is used to print the full path to the current directory.

clear
This command is used to clear the terminal of all text of previous commands and outputs.

ls
ls -a
ls -l
ls directory-name
This command stands for list and is used to list the contents of the current working directory (except for those that start with a dot, which are hidden files). To truly show all files the options -a can be applied. The option -l can be used with ls to print out the contents and include details about each file. If ls is given the name of a valid child directory it will list the contents of that child directory.

cd dir-to-access
cd dir-to-access/another-dir-to-access
cd ..
cd ../../..
cd ....
This command stands for change directory and is used to move between directories. The target directory must be within the current working directory. The name of the target should be placed after the command cd.

The use of double dots will move to the parent directory of the current one and multiple sets of dots may be used to move outwards multiple times. The last two examples are equivelent and move to the parent directory three times.

mkdir new-directory
mkdir new-directory/new-inner-directory
This command stands for make directory and is used to create new directories. The new directory is created in the current working directory. If a path is used to create a directory all but the last directory (the one to be created) should already exist.

cp file-to-copy name-of-copy
cp file-to-copy ../name-of-copy
cp file-to-copy other-dir/name-of-copy
cp -R directory-to-copy name-of-copy
This command stands for copy and is used to make a copy of a file. The first file given is the path to the file to be copied. The second is the path to the location where a copy should be made.

To copy a directory the option -R must be used as shown in the last example.

mv file-to-move new-file-name
mv file-to-move new-location/file-to-move
mv file-to-move new-location/new-file-name
This command stands for move and is used to move files from one location to another or rename files. In the first example the file will simply be renamed, in the second it will be moved into the new location with the same name. In the third it will be moved and renamed.

rm file-to-delete
rm -i file-to-delete
rm -d file-to-delete
rm -R directory-to-delete
rm -iR directory-to-delete
This command stands for remove and is used to remove / delete files. If you want to confirm the deletion the option -i can be used. This will not work on directories unless the option -R is used. This is also used to force the deletion of non-empty directories.

The options i and R can be used together (-iR) to recusivly move through the files in the target directory and decide if each one should be deleted.

rmdir directory-to-delete
This command stands for remove directory and is used to remove / delete directories. This will not work on files and cannot be used to force delete directories. To force deletion use rm -d name.

What Is Unix

UNIX is an operating system first developed in 1969 and first published in 1971. UNIX systems provides a simple set of tools which perform simple and well defined functions within a unified file system. They also use shell scripting to combine tools to perform complex actions.

UNIX operating systems are made up of three parts...

The Kernal: This is the central component of the operating system and performs multiple functions. Some of the subsystems of the kernal include process management, scheduling, file management, device management, network management, memory management and device interupt managment.

The Shell: This is an interface between user and kernal which presents as a CLI (command line interpreter). The user inputs commands and the shell starts the associated program. Once the program has run the CLI prompt the user for more input.

Processes and Programs

: The final part is the processes and programs that run to carry out actions on the system. Each excecuting program is given a unique PID (process identifier)
